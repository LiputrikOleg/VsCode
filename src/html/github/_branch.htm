Ветки

Общая информация

Ветвление позволяет вам иметь разные версии репозитория одновременно.

По умолчанию ваш репозиторий имеет одну ветку с именем main, которая считается окончательной веткой. Вы можете создавать дополнительные ветки от main в своем репозитории.

Ветвление полезно, когда вы хотите добавить новые функции в проект, не изменяя основной исходный код. Работа, выполненная в разных ветках, не будет отображаться в основной ветке, пока вы не объедините ее, о чем мы поговорим далее в этом руководстве. Вы можете использовать ветки для экспериментов и внесения изменений перед их фиксацией в main.

Когда вы создаете ветку от основной ветки, вы делаете копию или снимок main, какой она была на тот момент времени. Если кто-то другой внес изменения в основную ветку, пока вы работали над своей веткой, вы можете втянуть эти обновления.

На этой диаграмме показано:

  1. Основная ветка main
  2. Новая ветка с именем feature
  3. Путь, который проходит feature до ее слияния с main

Создание ветки

  1. Откройте репозиторий
  2. Над списком файлов нажмите на раскрывающееся меню с надписью main.
  3. Введите имя ветки, например: readme-edits , в текстовое поле.
  4. Нажмите «Create branch: readme-edits from main» .

Скриншот раскрывающегося списка веток для репозитория. «Create branch: readme-edits from „main“» выделено темно-оранжевым контуром.

Теперь у вас есть две ветки, main и readme-edits Сейчас они выглядят абсолютно одинаково. Далее вы добавите изменения в новую ветку readme-edits.

Фиксируем файл - Commit a file

Вы создали ветку!

Создание ветки позволяет редактировать проект без изменения основной ветки. Теперь, когда у вас есть ветка, пришло время создать файл и сделать первый коммит!

Что такое фиксация? Коммит — это набор изменений в файлах и папках вашего проекта. Коммит существует в ветке. Дополнительную информацию см. в разделе «О коммитах» .

Ваш первый коммит

Следующие шаги проведут вас через процесс внесения изменений на GitHub. В коммите фиксируются изменения при переименовании, изменении содержимого, создании нового файла и любых других изменениях, внесенных в ваш проект. В этом упражнении для фиксации изменения необходимо сначала добавить новый файл в новую ветку.

.md — это расширение файла, которое создает файл Markdown. Вы можете узнать больше о Markdown, посетив раздел «Основы синтаксиса написания и форматирования» в нашей документации или пройдя курс «Общение с использованием Markdown».

  1. Убедитесь, что вы находитесь на вашей Новой ветке.
  2. Выберите раскрывающийся список «Добавить файл» и нажмите «Создать новый файл».
  3. Скриншот
  4. В поле Имя вашего файла... введите PROFILE.md.
  5. В области «Введите сюда содержимое файла» скопируйте в файл следующее содержимое: Добро пожаловать в мой профиль на GitHub!
  6. Нажмите «Зафиксировать изменения...» в правом верхнем углу над полем содержимого.
  7. Скриншот
  8. Для коммитов вы можете ввести короткое сообщение о коммите, описывающее внесенные вами изменения. Это сообщение поможет другим узнать, что включено в вашу фиксацию. GitHub предлагает простое сообщение по умолчанию, но давайте немного изменим его для практики. Сначала введите «Добавить PROFILE.md» в первое поле ввода текста под названием «Зафиксировать сообщение».
  9. В этом уроке мы проигнорируем другие поля и нажмем «Зафиксировать изменения».
  10. Скриншот

Откройте запрос на извлечение - Pull request

Теперь, когда вы внесли изменения в проект и создали коммит, пришло время поделиться вашим предлагаемым изменением через запрос на извлечение!

Что такое запрос на извлечение?: Совместная работа происходит в запросе на извлечение. Запрос на извлечение показывает изменения в вашей ветке другим людям и позволяет им принимать, отклонять или предлагать дополнительные изменения в вашей ветке. В параллельном сравнении этот запрос на извлечение сохранит изменения, которые вы только что внесли в вашу ветку, и предложит применить их к основной ветке проекта. Для получения дополнительной информации о запросах на извлечение см. «О запросах на извлечение» .

Создайте запрос на извлечение

Вы могли заметить, что после вашего коммита отображается сообщение с указанием вашего недавнего push в вашей ветке и кнопкой с надписью Compare & pull request.

Объедините свой запрос на извлечение

Что такое объединение?: Объединение добавляет изменения в ваш запрос на извлечение и ветку в основную ветку. Для получения дополнительной информации о слияниях см. «Объединение запроса на извлечение» .

Как было отмечено в предыдущем шаге, вы могли видеть доказательства выполнения действий GitHub, которые автоматически продвигают ваши инструкции к следующему шагу. Вам придется подождать его завершения, прежде чем вы сможете объединить свой запрос на извлечение. Он будет готов, когда кнопка запроса на извлечение слияния станет зеленой.

Скриншот

Объедините запрос на извлечение

  1. Нажмите Merge pull request - «Объединить запрос на извлечение»
  2. Нажмите Confirm merge - «Подтвердить объединение».
  3. После объединения вашей ветки она вам больше не понадобится. Чтобы удалить эту ветку, нажмите «Удалить ветку».
